Select Query String Field

By default all standard fields are returned from search endpoints.
Note: Usually joined keys on joined tables are not shown by default as a "standard" field. They can be shown by setting an option on options query string field.

The fields return can be limited using the select query string field as JSON in the form: 
select={"field1":1,"field2":1}
The above would only return field1 and field2. It is also possible to supply a black list so that all fields are included except for those blacklisted with a zero: 
select={"field1":0,"field2":0}
This will all fields except for return field1 and field2.

In the case that query contains fields with same name on different tables, by default the field applies to first field in select list. To query on subsequent fields add table alias to search like: 
select={"field1":1,"table2.field1":1}
to be more explicit could write as: 
where={"table1.field1":1,"table2.field1":1}

It is also possible to white list complete tables like: 
select={"table1.*":1,"table2.*":1}
or to black list complete table like: 
select={"table1.*":0,"table2.*":0}
it is even possible to black list field in white listed table like: 
select={"table1.*":1,"table1.field1":0,,"table1.field2":0}
This would show all fields in table1 except for field1 and field2
